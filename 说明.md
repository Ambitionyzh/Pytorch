
🧠 PyTorch 深度学习核心基建脑图 (LLM 导向)
📦 一、 数据物流中心 (Data Pipeline)
Dataset (数据仓库)

定义数据的来源、总量 (__len__) 和提取单件货物的方法 (__getitem__)。

DataLoader (物流车队)

batch_size: 每次打包运送的数据量。

shuffle: 是否打乱顺序（训练集必须 True，测试集 False）。

⚡ 性能压榨与防坑 (num_workers)

原理: 开启多进程让 CPU 提前读图/预处理，防止 GPU “饿肚子”傻等。

⚠️ Windows 盖世神坑: Windows 采用 spawn 机制派发进程，会导致代码无限套娃重启。

💊 解药: 必须将核心执行代码包裹在 if __name__ == '__main__': 之下。

🏗️ 二、 模型构建工厂 (Model Architecture)
nn.Module (万物之源)

所有模型（从极简分类器到 GPT-4）必须继承的父类。

__init__: 采购清单（定义网络层、参数）。

forward: 组装图纸（定义数据流转路径）。

基础算子积木

nn.Linear: 全连接层（矩阵乘法），大模型的算力主力。

F.relu / nn.GELU: 激活函数，给网络注入非线性灵魂。

🔗 LLM 核心起步: nn.Embedding (词嵌入层)

作用: 将离散的文本 ID（如单词编号）映射为稠密的连续浮点数高维向量（魔法查表）。

输入限制: 必须是整数类型 (torch.long)。

⚙️ 三、 训练引擎与循环 (Training Engine)
三大件与设备搬运

Loss (误差尺子): MSE (回归), CrossEntropyLoss (分类/LLM预测下一个词)。

Optimizer (下山算法): SGD, Adam (自带动量的自适应学习率)。

.to(device): 铁律 —— 大厨（模型）和食材（数据）必须在同一个厨房（CPU/GPU）。

(注：PyTorch 完美支持 AMD 显卡，只需安装带有 ROCm 标签的专属版本即可)。

🔄 经典下山 5 步曲

optimizer.zero_grad(): 清空旧梯度（极易忘的 Bug 发源地）。

output = model(inputs): 前向传播。

loss = criterion(...): 计算误差。

loss.backward(): 反向传播，计算每个参数的梯度。

optimizer.step(): 更新权重参数。

🌍 四、 工业级分布式集群 (Distributed Training)
旧手推车: DataParallel (DP)

特点: 一行代码 nn.DataParallel(model)。

缺点: 单进程多线程，0 号主卡负责汇总梯度，容易过劳死，效率低。

高铁网络: DistributedDataParallel (DDP) [大厂标配]

特点: 多进程架构，每张卡完全独立，平起平坐。

基站建设: dist.init_process_group(backend='nccl') (N卡首选高速协议)。

发牌官: DistributedSampler (保证各卡分到的数据绝对不重样)。

🚀 集群启动器 (Launcher)

经典版: python -m torch.distributed.launch

现代版: torchrun

核心参数: --nproc_per_node (每台机器几张卡), --nnodes (总机器数)。

🩺 五、 性能法医与编译黑科技 (Optimization & PyTorch 2.0)
torch.profiler (性能 X 光机)

找耗时: aten::pow 等底层 C++ 算子耗时分析。

抓长文本瓶颈: 开启 record_shapes=True，追踪异常庞大的输入矩阵。

抓 OOM 爆显存: 开启 profile_memory=True，监控临时激活值带来的内存飙升。

torch.compile (PyTorch 2.0 一键起飞)

底层三剑客:

TorchDynamo: 前台大堂经理，捕获 Python 前向计算图。

AOTAutograd: 幕后精算师，提前推导反向图，优化显存和重计算逻辑。

TorchInductor: 底层大厨，生成极速 Triton/C++ 代码（算子融合）。

🚫 终极性能刺客: graph_break (图断裂)

成因: 数据依赖控制流 (if x>0)、Python 副作用 (print)。

后果: 强行中断 GPU 计算，退回 Python 解释器，导致编译加速失效。

⏱️ 跑分铁律 (Benchmarking)

必须要有 Warm-up (热身空跑)。

测 GPU 时间必须加 torch.cuda.synchronize() 强制同步。

(Windows CPU 环境需手动安装 cl.exe 编译器)。
